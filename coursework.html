1.2 Describing methods, skills and resources needed to complete digital projects successfully.

What do you need to make projects sucessful? When it comes to doing any form of software development whether it is a game or a website it is important to choose a design method to keep you on track during the creation process. The method that I would use is the waterfall model development methodology is a sequential design process which is often used in software development when a new project is being developed. The sequential design process is where the development of the project is done in order to get it completed efficiently and properly. An example of the waterfall model design process are the requirements, which is the step where the project is laid out on a document consisting of what needs to be included. The first step is where the discussions on what the project will look like takes place, the implementation process is where the first two steps requirements and design come together to actually create something solid and close to a final version of the software that is being created. The second step is the verification process which is where the project is then validated and by that I mean the software is then reviewed and inspected to make sure that it meets the requirements and design that was to be developed, and the final step is the maintenance stage where the project is reviewed again but this time to make sure that there aren’t any errors and if there are they can then be rectified.
